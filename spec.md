# spec.md — ひとり営業用 顧客管理アプリ 機能仕様書

---

## 1. 目的・利用シーン

### 目的
ひとり営業が Excel の顧客台帳で管理している情報を、ブラウザ 1 枚で扱えるようにする。顧客と商談を登録・一覧・編集・削除でき、見込み → 提案 → 成約 のパイプラインを 3 列のカンバンで見渡せる、最小構成の CRM。業務用 CRM SaaS（Salesforce / HubSpot 等）の核機能だけを抜き出した学習用デモに相当する。

### 想定ユーザー
- 1 人で営業活動を行っているフリーランス・個人事業主
- Excel 管理に限界を感じているが、SaaS 導入はオーバースペックな規模感

### 利用シーン
- 朝の業務開始時に商談の進捗を俯瞰する（パイプラインビュー）
- 打ち合わせ直前に顧客情報を確認する（顧客ビュー＋検索）
- 商談後にフォローアップメモを更新する（商談フォーム）
- ステータスが進んだ商談を 1 クリックで次の列に送る（パイプラインの「→」）

## 2. データ項目

### 顧客 Customer

| 項目 | キー名 | 必須 | 型 | 備考 |
|---|---|---|---|---|
| ID | `id` | ○ | string | 自動採番 |
| 会社名 | `company` | ○ | string | 例: 株式会社サンプル |
| 担当者名 | `contact` | ○ | string | 例: 山田 太郎 |
| 役職 | `title` | - | string | 任意 |
| メール | `email` | - | string | 任意 |
| 電話 | `phone` | - | string | 任意 |
| メモ | `memo` | - | string | 複数行可 |
| 作成日時 | `createdAt` | ○ | string (ISO) | 自動付与 |

### 商談 Deal

| 項目 | キー名 | 必須 | 型 | 備考 |
|---|---|---|---|---|
| ID | `id` | ○ | string | 自動採番 |
| 顧客ID | `customerId` | ○ | string | 顧客と 1 対多で紐付け |
| タイトル | `title` | ○ | string | 例: サービスA導入提案 |
| 金額 | `amount` | - | number\|null | 整数、円 |
| ステータス | `status` | ○ | `'lead'\|'proposal'\|'won'` | 見込み / 提案 / 成約 |
| フォローアップメモ | `followup` | - | string | 複数行可 |
| 作成日時 | `createdAt` | ○ | string (ISO) | 自動付与 |
| 更新日時 | `updatedAt` | ○ | string (ISO) | 保存のたびに更新 |

### 1 対多の関係

- 1 人の **顧客** に対して、複数の **商談** が紐付く
- Deal 側が `customerId` を外部キーとして持つ
- Customer 側は商談 ID のリストを持たない（正規化維持）

```
Customer (1) ───< Deal (N)
           customerId
```

## 3. 画面構成

### ビュー切替（上部タブ）

画面最上部にタブを 2 つ置く:

| タブ | 表示対象 |
|---|---|
| 顧客 | 左ペイン（顧客リスト）＋ 右ペイン（詳細 / フォーム） |
| パイプライン | 3 列カンバン |

選択中のタブはアクセントカラー (`#c15f3c`) で強調、未選択は `text-gray-500`。

---

### ビュー① 顧客（ツーカラム）

#### 左ペイン（幅 340px 固定）

上から順に:
1. 検索ボックス（会社名・担当者名・役職を横断検索）
2. 「新規顧客」ボタン（プライマリ）
3. 顧客カードの縦リスト（作成日時の新しい順）

顧客カード 1 件に表示する情報:
- 会社名（太字）
- 担当者名＋役職（小さめ・グレー）

選択中のカードは背景色と左アクセントラインでハイライト。

#### 右ペイン（flex-1）: 4 モード

右ペインは以下 4 モードのいずれかを表示。

| モード | ID | 表示タイミング |
|---|---|---|
| 空状態 | `pane-empty` | 初期表示 / 顧客削除後 |
| 顧客詳細 | `pane-detail` | 左リストで顧客を選択したとき |
| 顧客フォーム | `pane-customer-form` | 「新規顧客」または詳細の「編集」クリック時 |
| 商談フォーム | `pane-deal-form` | 詳細の「商談を追加」または商談行クリック時 |

##### 顧客詳細 `pane-detail`
- 連絡先ブロック（会社名・担当者・役職・メール・電話・メモ）
- 「編集」「削除」ボタン
- 商談一覧（各行: タイトル＋ステータスバッジ＋金額）
- 「商談を追加」ボタン
- 商談行クリックで商談フォーム（編集）へ遷移

##### 顧客フォーム `pane-customer-form`
- 全項目の入力欄（必須項目は `company` / `contact`）
- 「保存」「キャンセル」ボタン
- 保存後は顧客詳細モードに戻る

##### 商談フォーム `pane-deal-form`
- タイトル・金額・ステータス（セレクト）・フォローアップメモ
- 「保存」「キャンセル」ボタン、編集時のみ「削除」ボタン
- 保存後は顧客詳細モードに戻る

---

### ビュー② パイプライン（カンバン 3 列）

3 列を等幅 (`grid-cols-3`) で左から配置:

| 列 | ステータス | 色（見出し・バッジ） |
|---|---|---|
| 見込み | `lead` | グレー |
| 提案 | `proposal` | オレンジ |
| 成約 | `won` | グリーン |

各カードに表示する情報:
- 商談タイトル（太字）
- 紐付く顧客の会社名（グレー）
- 金額（右寄せ、未設定なら「—」）
- 右下に `←` / `→` ボタン
  - 先頭列（見込み）では `←` を非表示
  - 末尾列（成約）では `→` を非表示
  - クリックで隣のステータスに直接更新（フォーム経由せず `updatedAt` を更新）
- カード本体クリックで「顧客」ビューに戻り、該当顧客の商談フォーム（編集）を開く

モバイル対応なし（PC 表示のみ）。

## 4. CRUD 操作フロー

### 顧客 Customer

| 操作 | 起点 | フロー | 終点 |
|---|---|---|---|
| Create | 左ペイン「新規顧客」ボタン | 顧客フォーム → 保存 → localStorage に追加 | 顧客詳細 |
| Read (一覧) | 起動時 | `crm-customers` を読み込み → 作成日時降順で左ペインに表示 | 左ペイン |
| Read (検索) | 検索ボックス入力 | 会社名・担当者名・役職を横断で前方／部分一致 | 左ペインを動的に絞り込み |
| Update | 顧客詳細の「編集」ボタン | 顧客フォーム（既存値入り）→ 保存 | 顧客詳細 |
| Delete | 顧客詳細の「削除」ボタン | 確認ダイアログ → 顧客削除 → 紐付く商談を連鎖削除 | 右ペイン空状態 |

### 商談 Deal

| 操作 | 起点 | フロー | 終点 |
|---|---|---|---|
| Create | 顧客詳細の「商談を追加」ボタン | 商談フォーム（新規）→ 保存 | 顧客詳細（商談一覧に反映） |
| Read | 顧客詳細の商談一覧 / パイプラインビュー | それぞれのビューに応じて描画 | - |
| Update | 顧客詳細の商談行クリック / パイプラインのカードクリック | 商談フォーム（既存値入り）→ 保存 → `updatedAt` 更新 | 顧客詳細 |
| Delete | 商談フォームの「削除」ボタン | 確認ダイアログ → 商談削除 | 顧客詳細 |
| Status 遷移（簡易） | パイプラインのカード `←` / `→` ボタン | フォームを経由せず status を隣に変更 → `updatedAt` 更新 | パイプラインを再描画 |

## 5. 永続化方針

### localStorage キー

| キー | 形式 | 中身 |
|---|---|---|
| `crm-customers` | `JSON.stringify(Customer[])` | 顧客の配列 |
| `crm-deals` | `JSON.stringify(Deal[])` | 商談の配列 |

### 保存タイミング
- 顧客・商談の Create / Update / Delete のたびに、該当キーを **全件書き直し**（差分更新はしない）
- パイプラインの `←` / `→` ボタンによるステータス遷移時も `crm-deals` を全件書き直し、`updatedAt` を更新

### 顧客削除時の連鎖削除ルール
顧客 `X` を削除するとき、以下を 1 トランザクション相当で実行する:

1. `crm-customers` から `id === X.id` を除外して書き直す
2. `crm-deals` から `customerId === X.id` の商談を **すべて除外** して書き直す
3. 右ペインを空状態に戻す

順序は必ず **顧客 → 商談** の順。どちらか一方だけ失敗した場合でも、関連商談が孤児（orphan）になることを避けるため、2 の処理はスキップしない。

### 読み込みタイミング
- 起動時に 1 回だけ `crm-customers` / `crm-deals` を読む
- 以降はメモリ上の配列を正とし、保存時に localStorage へ書き戻す

## 7. 対象外の機能（今回作らない）

以下は意図的にスコープから外す。将来やるとしても、別タスクで扱う。

| 機能 | 代替・理由 |
|---|---|
| ドラッグ＆ドロップでのステータス移動 | パイプラインの `←` / `→` ボタンで代替 |
| クラウド保存・他端末共有 | localStorage のみ（単一ブラウザ前提） |
| ログイン・マルチユーザー | 単一ユーザー前提 |
| CSV / JSON のインポート・エクスポート | 初期データはハードコード |
| モバイル対応（レスポンシブ） | PC 表示のみ |
| メール送信連携 | 外部API 禁止のため |
| カレンダー連携 | 外部API 禁止のため |
| 並び替え・ソート UI | 作成日時降順固定 |
| 検索の高度化（AND/OR・タグ） | 単純な部分一致のみ |
